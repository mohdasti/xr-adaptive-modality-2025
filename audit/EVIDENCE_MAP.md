| `docs/ADAPTATION_POLICY_ROOT_CAUSE.md` | Root cause analysis for width inflation bug | Documents that policy emitted actions but UI did not apply | **Authoritative** for bug diagnosis |

**Key logic:** `policy.ts` line 375–383: if `pressure_only && !pressureEnabled` → returns `action: 'none'`. Width inflation requires pressure enabled. Event bus: `policy:change` → TaskPane `handlePolicyChange` → `setWidthScale`. FittsTask receives `widthScale` and computes `effectiveWidth = config.W * widthScale`.
